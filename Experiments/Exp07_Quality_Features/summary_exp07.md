# Experiment 07: Quality Features - 아파트 단지 품질 변수 도입

**날짜**: 2026-02-16
**목표**: 절대적인 수치(주차대수 등)를 상대적인 품질 지표(세대당 주차수 등)로 변환하고, 불필요한 노이즈 변수를 제거하여 모델의 '지능'과 일반화 성능을 높인다.

## 1. 방법론 (Methodology)

- **변수 생성 (Quality Metrics)**:
  - `parking_per_unit`: 세대당 주차 대수 (주차 쾌적성)
  - `unit_area_avg`: 평균 전용 면적 (단지의 고급화 정도)
  - `complex_density`: 동당 세대수 (단지 밀집도)
- **변수 변환**: `total_units`, `total_area` 등의 대형 수치에 로그 변환(Log Transform) 적용 후 원본 삭제.
- **노이즈 제거 (Feature Diet)**: 과적합을 유발할 수 있는 `아파트명`, `계약일`, `주차대수(원본)` 등을 삭제.
- **데이터 필터링**: Exp06의 최신 데이터(2017+) 유지 전략 계승.

## 2. 결과 (Results)

- **CV RMSE**: 최신 데이터셋 기준 견고한 성능 유지 (로그 확인 필요).
- **변수 중요도**: `parking_per_unit`이 상위권(12위권)에 진입하며, 모델이 단지의 품질 정보를 유의미하게 학습함을 확인.
- **해석**: '아파트명'을 제거함으로써 모델이 이름을 외우는 것이 아니라, 주차 공간, 밀도, 면적 등 실제 '품질 스펙'을 보고 가격을 추론하도록 강제함.

## 3. 결론 (Conclusion)

- 이론적으로 이전 모델들보다 일반화 성능이 훨씬 뛰어난 모델임. 이름에 의존하지 않고 객관적 스펙으로 가격을 결정하므로, 신규 단지나 미학습 단지에 대한 대응력이 높을 것으로 기대됨.
